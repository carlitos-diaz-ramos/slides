Introducción

Este documento recolle os aspectos fundamentais para preparar unha presentación en html.

Está concevido como manual de instruccións simplificado, ten exemplos de código, e debería de verse o propio ficheiro como un exemplo de como xerar unha presentación.

Non obstante, o documento en si mesmo non pretende ser un exemplo de como debe organizarse unha presentación. O tipo de letra elexido é posiblemente un pouco pequeno, así como o espaciado.

O elemento <head>

O código mínimo que debe ir no preámbulo do documento é:

<head>
<meta charset="utf-8">
<meta name="viewport" content="width=device-width, initial-scale=1">

<link rel="stylesheet" href="code/html5-reset.css">
<link rel="stylesheet" href="code/slides.css">
<!-- Include the following stylesheet for printing: -->
<!-- <link rel="stylesheet" href="code/print.css"> -->
<!-- Or this one for avoiding transitions: -->
<!-- <link rel="stylesheet" media="all" href="code/notransitions.css"> -->
<script src="code/slides.js"></script>
</head>

O arquivo slides.css é responsable do estilo básico da presentación. Pódense engadir outros arquivos de estilo a maiores.

O arquivo slides.js ten o código para facer as transicións.

En caso de querer imprimir, cambia-lo arquivo slide.css por print.css. Para imprimir sen transicións empregar en cambio notransitions.css

Diapositivas

O documento estará comprendido nunha etiqueta <main>, e cada diapositiva será representada por un <article>.

Para moverse na presentación:

  • ↩, ␣, →: paso seguinte.
  • ←: paso anterior.
  • ⇟: diapositiva seguinte.
  • ⇞: diapositiva anterior.
  • ↖: táboa de contidos.
  • ⌃+↖: primeira diapositiva.
  • ↘: última diapositiva.
  • Movemento da roda do rato: permite moverse cara adiante e atrás.

Nas trasparencias seguintes preséntanse as plantillas habituais para facer diapositivas.

Plantilla

En xeral, salvo en casos como a diapositiva do título, a plantilla para unha diapositiva é como segure:

<article>
<header>
<h1>Título</h1>
</header>

<section>
<!-- Aquí vai o contido -->
</section>
</article>

Cada <article> representa unha diapositiva. No <h1> situado en <header> vai o seu título, que por defecto se ve como unha cinta. O contido da diapositiva vai dentro de <section>.

Na barra de título insértanse automaticamente tres botóns para ir adiante, atrás, e ós contidos.

Diapositiva de título

Plantilla para a diapositiva do título:

<article id="title">
<section>
<h1>Título da presentación</h1>
<h2>Subtítulo</h2>
<h3>Subsubtítulo</h3>

<footer>
<img src="pics/logo.jpg" alt="un logo">
<button type="button" class="button-next">&#10095;</button>
<ul>
<li>Autor do traballo</li>
<li>Afiliación</li>
<li><time datetime="2019-10-31">Data que sexa</time></li>
</ul>
</footer>
</section>
</article>

Diapositiva de contidos

Plantilla para táboa de contidos:

<article id="contents">
<header>
<h1>Contidos</h1>
</header>

<section>
<h1><a href="#link-seccion-1">Sección 1</a></h1>
<h2><a href="#link-subseccion-1-1">Subsección 1.1</a></h2>
<h2><a href="#link-subseccion-1-2">Subsección 1.2</a></h2>
<h1><a href="#link-seccion-2">Sección 2</a></h1>
<h2><a href="#link-subseccion-2-1">Subsección 2.1</a></h2>
<h2><a href="#etc">Etc.</a></h2>
</section>
</article>

As diapositivas que se queiran enlazar teñen que ter un id que é o que se pon en cada href.

SVG

Neste arquivo html inclúese ó final un <svg> invisible onde se definen frechas que poden ser empregadas para presentacións.

<figure class="row-pictures">
<svg><use href="#right-arrow"/></svg>
<svg><use href="#left-right-arrow" class="bg-yellow"/></svg>
<svg><use href="#down-arrow" class="bg-green"/></svg>
<svg><use href="#implies" class="bg-white"/></svg>
<svg><use href="#iff"/></svg>
</figure>

Como se ve no código anterior, algunhas das frechas foron combinadas con algunha cor de fondo.

Recórdese que incorporar debuxos feitos en svg dende arquivos externos presenta dificultades se queremos cambia-la súa apariencia mediante css, xa que nese caso habería que executa-lo ficheiro a través dun servidor. Resulta por tanto máis cómodo definir eses gráficos ó final do arquivo html fóra da etiqueta <main>

Imaxes

Nas diapositivas que seguen preséntanse algúns entornos predefinidos que permiten colocar imaxes nunha presentación.

A variable --section-display ten como valor por defecto flex, o cal significa que non se colapsan marxes, e no caso das figuras, estas non poden ser flotadas.

Como exemplo, nesta diapositiva escribiuse <article id="imaxes"> e <figure class="picture-side">xunto co código css:

#imaxes {
    --section-display: block;
}
figure.picture-side {
    float: right;
    width: 30%;
}

Animacións

Podémonos mover cara adiante coa frecha dereita, retorno, ou barra espaciadora, e cara atrás coa frecha esquerda. Ademais tamén é posible ir facendo aparecer paulatinamente os elementos dunha diapositiva. Esta diapositiva presenta un exemplo de tal cousa xunto co código necesario.

Este aparece segundo

<div class="bg-blue show-20">
<p>Este aparece segundo</p>
</div>

Este aparece primeiro

<div class="bg-green show-10">
<p>Este aparece primeiro</p>
</div>

O efecto de aparecer consíguese engadindo á clase do elemento correspondente show-número, onde o número indica a orde de aparición. É boa idea poñe-los números de dez en dez (como en BASIC) por se máis tarde decidimos intercalar algún.

Animacións 2

Por defecto, os elementos que irán aparecendo vense difuminados. Se por exemplo queremos que estean ocultos e aparezan inmediatamente simplemente se poría:

[class*="show-"] {
    visibility: hidden;
}
.shown {
    visibility: visible;
}

Animacións 3

Ademais de aparecer, tamén se poden facer outras animacións con animate-número. Para que isto funcione hai que especifica-lo estilo explicitamente en css.

Por exemplo, este parágrafo primeiro aparece e despois ponse en vermello xunto con outro elemento do parágrafo anterior.

Para conseguir ese efecto, no parágrafo anterior ponse o código: <p class="animation-example show-10 animate-20">, xunto co css:

.animation-example.animated {
    color: red;
}

Esta característica aínda é experimental.

Animacións 4

Tamén se poden facer desaparecer elementos dunha diapositiva mediante a clase erase-número. Por defecto, está programada a clase .erased que ten como estilo visibility: hidden. Por tanto, aínda que o elemento desapareza, o seu espacio segue sendo ocupado.

Como exemplo, no fondo desta diapositiva faise desaparecer un debuxo. Nesta diapositiva o elemento foi colocado con position: absolute para que non inflúa no fluxo da diapositiva. Por esa razón, esta diapositiva tamén ten se coloca con display: block.

Esta característica aínda é experimental.

Animacións 5

A continuación preséntase un exemplo no que unha figura se intercambia por outra (faise desaparecer unha e aparece-la outra ó mesmo tempo). A parte relevante do código é:

<div class="row-pictures erased-example show-10"> ... </div> <div class="row-pictures erased-example show-10"> ... </div>

Con css:

.erased-example {
    position: absolute;
    bottom: 5em;
    right: 2em;
}
[class*="show-"].erased-example {
    opacity: 0;
}
.shown.erased-example {
    opacity: 1;
}

Cambia-los estilos 1

Os estilos definidos en slides.css poden ser modificados cargando unha folla de estilo. Por exemplo, nesta presentación cárgase o arquivo custom.css que, entre outras cousas, inserta a imaxe de fondo.

article {
    background-image: url("pics/berndt-difuse.png");
    background-position: 10vh 12vh;
    background-repeat: no-repeat;
    background-size: 75vh auto;
}

Non obstante, parte dos estilos están controlados simplemente por variables que só habería que modificar. Como exemplo, a cor de letra desta presentación non é a cor black por defecto senón:

:root {
    --text-color: hsl(0, 0%, 20%);
}

Outro exemplo de cambio de estilos pode verse en dark.css que implementa o modo escuro.

Relación de aspecto

O parámetro máis importante que pode ser necesario cambiar é a relación de aspecto --screen-width. Os valores típicos a tomar son:

  • relación de aspecto 4:3: --screen-width: 133vh.
  • relación de aspecto 16:9: --screen-width: 177vh.
  • relación de aspecto 16:10: --screen-width: 160vh.

Convén tamén cambiar --side-margin se a relación non é 4:3 para evitar liñas moi longas.

Maquetado da páxina

Como xa vimos anteriormente ó falar de imaxes, o maquetado da páxina faise mediante as dúas variables:

    --section-display: flex;
    --vertical-justify: flex-start;

Os valores por defecto amosados teñen certos problemas.

En primeiro lugar non se colapsan as marxes nin se permite flotar elementos. Para permitir isto habería que empregar --section-display: block. É posible que entón tamén haxa que cambia-lo valor de separación entre parágrafos --par-sep para axusta-lo espaciado.

Por outro lado, o valor por defecto alinea verticalmente os elementos á parte superior. Podería empregarse en --vertical-justify os valores space-around, space-between ou space-evenly.

Nótese en todo caso que non é necesario cambiar este valor globalmente, senón que podería facerse só para unha diapositiva ou un grupo delas mediante un id, class, ou un selector axeitado.

Variables css 1

A continuación presentamos unha lista de variables que poden ser modificadas, xunto co seu valor por defecto á hora de escribir este documento:

  • Color de fondo das diapositivas: --bg-color: white
  • Color do texto: --text-color: black
  • Tamaño da fonte principal: --text-size: 4vh
  • Cor do texto enfatizado fortemente: --strong-color: red
  • Cor das fórmulas matemáticas: --math-color: purple
  • Cor do código informático: --code-color: indigo
  • Marxe superior do contido principal: --sec-margin-top: 1rem
  • Marxe lateral do contido principal: --side-margin: 2em
  • Separación entre párrafos: --par-sep: 0.3em
  • Modo de maquetación: --section-display: flex
  • Xustificación vertical: --vertical-justify: flex-start

Variables css 2

Continuación:

  • Cor dos links: --a-link: brown;
  • Cor dos links visitados: --a-visited: purple;
  • Cor dos links ó pasar porriba: --a-hover: chocolate;
  • Cor do título: --title-color: maroon;
  • Cor da cinta da cabeceira: --header-color: linear-gradient(to bottom right, black, blue);
  • Cor do texto da cabeceira: --header-text-color: white;
  • Altura da cinta da cabeceira: --header-height: 2.2em;
  • Marxe lateral dos entornos e caixas: --box-side-margin: -0.3em;
  • Sombra do borde (para entornos e caixas): --shadow: 0 0.15rem 0.3rem 0 lightgrey, 0 0.2rem 0.7rem 0 lightgrey;
  • Radio dos bordes: --border: 0.3rem;

Variables css 4

Para determina-las cores das caixas basta con da-las cores inicial e final dun gradiente e a cor sólida que se necesitaría se non houbese gradiente (por exemplo necesario para as frechas). Por exemplo, as caixas azuis ou os problemas empregan esta combinación.

.problem, .bg-blue {
    --color-start: hsl(180, 100%, 97%);
    --color-end: hsl(195, 53%, 79%);
    --color-solid: lightblue;
}

Para modificar unha caixa simplemente se cambiaría o código análogo ó de arriba. En caso de querer definir un novo tipo, ademais dese código, habería que poñer por exemplo:

.nova-caixa {
    background: linear-gradient(
        to bottom, var(--color-start), var(--color-end));
}

Imprimir diapositivas

Para imprimir a pdf:

  • Seleccionar (descomentar na plantilla) o arquivo css code/print.css.
  • Seleccionar "Imprimir co cadro de diálogo de sistema" e en "Adobe" ir a "Preferencias". En "Adobe PDF Page Size" darlle a "Add..." e crear un tamaño da proporción necesaria (p. ex. 320mm x 200mm para 16:10). Salvar.
  • A seguinte vez que se imprima aparecerá ese tamaño, e imprimir normal.

En vez de code/print.css pódese coller code/notransitions.css. Con este método non se conservan as animacións, o cal é bo para distribuí-la presentación.
